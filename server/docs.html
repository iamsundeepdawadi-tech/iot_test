1. System Overview

IoT Farm is a bidirectional IoT platform for monitoring sensors and controlling actuators. Built with ESP32 + vanilla PHP + MySQL.

  • ESP32 reads sensors, sends JSON data to server via HTTP POST
  • PHP Server stores data in per-device MySQL tables
  • Config-based control вҖ” ESP32 polls /config/{device_id} for control signals (no MQTT needed)
  • Dashboard вҖ” AJAX-based, auto-refreshing, reactive to device type
  • Code Generator вҖ” generates ready-to-paste Arduino IDE code per device

Key Design Decisions

  • Config-based control instead of command queues вҖ” simpler, ESP32 just polls current state, no ack needed
  • Per-device tables вҖ” each device has its own data_{device_id} table, so different device types can have different JSON schemas
  • Device types as metadata вҖ” dashboard renders dynamically from the TYPES JS object, making it trivial to add new types
  • No MQTT вҖ” pure HTTP REST, works through any firewall/NAT
  • No frameworks вҖ” vanilla PHP + vanilla JS, zero dependencies on server

Go to Dashboard, select your device. Live data appears within seconds. Use controls to toggle LED/relays.

ISR Best Practices

  • Use ARDUINO_ISR_ATTR attribute on ISR functions
  • Mark shared variables as volatile
  • Keep ISRs short вҖ” set flags only, handle in loop()
  • Use millis() instead of delay() for non-blocking timing
  • Don't call Serial.println() inside ISRs

9. Troubleshooting

ESP32 can't connect to WiFi

  • Check SSID and password are exactly correct (case-sensitive)
  • Make sure ESP32 is within WiFi range
  • Open Serial Monitor (115200 baud) to see connection attempts
  • Try power cycling the ESP32

ESP32 connects but server returns errors

  • Verify the device is registered on the dashboard first
  • Check the SERVER_URL in the Arduino code matches your domain
  • Test with curl: curl https://iot.aitalim.com/api/devices.php
  • Check .htaccess is uploaded (needed for clean URLs)
  • Ensure mod_rewrite is enabled on Apache (cPanel has it by default)

Dashboard shows no data

  • Select a device from the dropdown
  • Check the device's data table exists in phpMyAdmin
  • Try sending test data: curl -X POST .../data/device_id -d '{"value":42}'

Config changes not reaching ESP32

  • ESP32 polls config every 3-5 seconds вҖ” wait a moment
  • Check Serial Monitor for [Config] log lines
  • Verify config with: curl https://iot.aitalim.com/config/device_id

GitHub Actions deploy failing

  • Verify all 3 secrets are set: FTP_SERVER, FTP_USERNAME, FTP_PASSWORD
  • Check the FTP path in deploy.yml matches your subdomain directory
  • Go to Actions tab on GitHub to see error logs
