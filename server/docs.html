What is IoT Farm?

IoT Farm is a lightweight, open-source platform for connecting ESP32 microcontrollers to a web dashboard. It runs entirely on Vanilla PHP + MySQL — no frameworks, no MQTT broker, no Docker. Just upload to any cPanel hosting and it works.

How It Works

ESP32 Device PHP Server (cPanel) ┌──────────────┐ ┌──────────────────┐ │ Read Sensor │──POST /data/id──→ │ Store in MySQL │ │ (DHT22 etc) │ │ (per-device tbl)│ │ │──GET /config/id──→ │ Return config │ │ Apply relay │←─── JSON ──────── │ (light, heater) │ └──────────────┘ └──────────────────┘ ↑ Browser Dashboard ─────── AJAX ─────────────┘ (index.html) (every 5s)

Quick Start (5 Minutes)

Step 1: Register a Device

  1. Go to Devices → + Add New tab
  2. Select a device type (start with random_test for testing)
  3. Enter a Device ID (e.g., my_first_device)
  4. Enter a Name and Location
  5. Click Register Device

Step 2: Get the Arduino Code

  1. On the device card, click View Code
  2. Click Copy Code or Download .ino
  3. Paste into Arduino IDE

Step 3: Flash the ESP32

  1. Open Arduino IDE
  2. Install board: Tools → Board Manager → search "ESP32"
  3. Select board: Tools → Board → ESP32 Dev Module (or TTGO T-Call)
  4. Select port: Tools → Port → /dev/cu.usbserial-*
  5. Click Upload (→ arrow button)
  6. Open Serial Monitor (115200 baud) to see WiFi connection and data sending

Step 4: View Live Data

  1. Go to Dashboard
  2. Select your device from the dropdown
  3. Watch live sensor readings update every 5 seconds

No ESP32? Use the Python Emulator

python3 server_python_test/test_device.py

This emulates a Chicken Coop device, sending random temperature/humidity data to the live server. Great for testing without hardware.

ESP32 Hardware Guide

What You Need

  • ESP32 Board — Any ESP32 board works. We use a TTGO T-Call ESP32 SIM800L.
  • USB Cable — Micro-USB or USB-C depending on your board
  • WiFi Network — Currently: accounthack_fbnpa_2 / CLED02502F
  • Sensors (optional) — DHT22, Soil Moisture sensor, LDR, etc.

Arduino IDE Setup

  1. Download Arduino IDE (v2.x recommended)
  2. Go to File → Preferences
  3. Add this URL to "Additional Board Manager URLs":
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  1. Go to Tools → Board Manager, search for esp32, and install esp32 by Espressif
  2. Install required libraries via Sketch → Include Library → Manage Libraries:
    • ArduinoJson (by Benoit Blanchon)
    • DHT sensor library (by Adafruit) — for chicken_coop type

TTGO T-Call Safe Pins

Important: The TTGO T-Call uses some GPIO pins for the SIM800L modem. Only use these safe GPIO pins for sensors and relays:
PinSafe?Suggested Use
GPIO 2✓Built-in LED / general output
GPIO 13✓Relay 1 (Light)
GPIO 14✓Relay 2 (Heater)
GPIO 15✓General I/O
GPIO 25✓DHT22 Data Pin
GPIO 32✓Analog Input (Soil Moisture)
GPIO 33✓Analog Input (LDR)
GPIO 4✗ (SIM800L)Do NOT use
GPIO 5✗ (SIM800L)Do NOT use
GPIO 26/27✗ (SIM800L)Do NOT use

Wiring Example: DHT22 Sensor

DHT22 Pin 1 (VCC) → ESP32 3.3V DHT22 Pin 2 (DATA) → ESP32 GPIO 25 (with 10kΩ pull-up to 3.3V) DHT22 Pin 3 (NC) → Not connected DHT22 Pin 4 (GND) → ESP32 GND

Wiring Example: Relay Module

Relay VCC → ESP32 5V (or Vin) Relay GND → ESP32 GND Relay IN1 → ESP32 GPIO 13 (Light) Relay IN2 → ESP32 GPIO 14 (Heater)

How ESP32 Code Works

The generated Arduino code follows a simple pattern:

  1. setup() — Connect to WiFi, initialize sensors and pins
  2. loop() — Every 10 seconds: read sensors → POST JSON to /data/{device_id}
  3. loop() — Every 5 seconds: GET config from /config/{device_id} → apply relay states
// Simplified flow: void loop() { if (now - lastSend >= 10000) { // Read DHT22 → POST to /data/my_device sendData(); } if (now - lastConfig >= 5000) { // GET /config/my_device → apply relays readConfig(); } }

POST /api/devices.php Content-Type: application/json { "device_id": "my_new_device", "device_type": "chicken_coop", "name": "Barn B Sensor", "location": "Barn B" }

Valid types: chicken_coop, random_test, soil_sensor, ldr_sensor

Sensor Data

Send Data (ESP32 → Server)

POST /data/{device_id} Content-Type: application/json {"temp_c": 25.3, "humidity": 62.1}

Read Data (Dashboard → Server)

GET /data/{device_id}?limit=50

Response:

[ {"id": 42, "recorded_at": "2026-03-18 12:00:05", "data": {"temp_c": 25.3, "humidity": 62.1}}, {"id": 41, "recorded_at": "2026-03-18 11:59:55", "data": {"temp_c": 25.1, "humidity": 62.0}} ]

Device Config

Read Config (ESP32 Polls This)

GET /config/{device_id}

Response:

{"light": "on", "heater": "off", "desired_temp": 28}

Update Config (Dashboard Sends This)

POST /config/{device_id} Content-Type: application/json {"light": "on"}

Partial updates are supported — only the keys you send are updated.

Test with curl

# List all devices curl https://iot.aitalim.com/api/devices.php # Register a device curl -X POST https://iot.aitalim.com/api/devices.php \ -H "Content-Type: application/json" \ -d '{"device_id":"curl_test","device_type":"random_test","name":"Curl Test","location":"CLI"}' # Send data curl -X POST https://iot.aitalim.com/data/curl_test \ -H "Content-Type: application/json" \ -d '{"value": 42}' # Read config curl https://iot.aitalim.com/config/curl_test # Toggle LED on curl -X POST https://iot.aitalim.com/config/curl_test \ -H "Content-Type: application/json" \ -d '{"led": "on"}'

Developer Guide — Adding a New Device Type

The modular architecture makes adding a new device type incredibly simple. You only need to create one JavaScript file and update two lines in existing files.

Step 1: Create the Module File

Create a new file at js/types/your_device.js:

/** * Your Device Module * File: js/types/water_pump.js */ import { WIFI_SSID, WIFI_PASS, SERVER_URL } from '../core.js'; // 1. Display name and description (shown in device registration form) export const label = 'Water Pump Controller'; export const desc = 'Controls a water pump with soil moisture feedback.'; // 2. Define what sensors this device sends // Keys must match the JSON keys your ESP32 POSTs export const sensors = { soil_moisture: { label: 'Soil Moisture', unit: '%' }, pump_runtime: { label: 'Pump Runtime', unit: 's' } }; // 3. Define dashboard controls (toggle, range, or leave empty []) export const controls = [ { key: 'pump', label: 'Water Pump', type: 'toggle' }, { key: 'threshold', label: 'Moisture Threshold (%)', type: 'range', min: 0, max: 100, step: 5 } ]; // 4. Default config values (stored in DB on registration) export const configDefaults = { pump: 'off', threshold: 30 }; // 5. Generate the Arduino code for this device export function generateCode(d) { return `// ${d.name} — Water Pump Controller #include <WiFi.h> #include <HTTPClient.h> #include <ArduinoJson.h> #define WIFI_SSID "${WIFI_SSID}" #define WIFI_PASSWORD "${WIFI_PASS}" #define SERVER_URL "${SERVER_URL}" #define DEVICE_ID "${d.device_id}" #define MOISTURE_PIN 32 #define PUMP_PIN 13 unsigned long lastSend = 0; unsigned long lastConfig = 0; void setup() { Serial.begin(115200); pinMode(PUMP_PIN, OUTPUT); WiFi.begin(WIFI_SSID, WIFI_PASSWORD); while (WiFi.status() != WL_CONNECTED) delay(500); } void loop() { unsigned long now = millis(); if (now - lastSend >= 10000) { lastSend = now; sendData(); } if (now - lastConfig >= 5000) { lastConfig = now; readConfig(); } delay(100); } void sendData() { int raw = analogRead(MOISTURE_PIN); int moisture = map(raw, 4095, 0, 0, 100); JsonDocument doc; doc["soil_moisture"] = moisture; doc["pump_runtime"] = 0; String payload; serializeJson(doc, payload); HTTPClient http; http.begin(String(SERVER_URL) + "/data/" + DEVICE_ID); http.addHeader("Content-Type", "application/json"); http.POST(payload); http.end(); } void readConfig() { HTTPClient http; http.begin(String(SERVER_URL) + "/config/" + DEVICE_ID); if (http.GET() == 200) { JsonDocument doc; deserializeJson(doc, http.getString()); const char* pump = doc["pump"] | "off"; digitalWrite(PUMP_PIN, strcmp(pump, "on") == 0 ? HIGH : LOW); } http.end(); }`; }

Step 2: Register the Type in the Backend

Open api/devices.php and add your type to the $defaultConfigs array:

$defaultConfigs = [ 'chicken_coop' => ['light' => 'off', 'heater' => 'off', 'desired_temp' => 28], 'soil_sensor' => [], 'ldr_sensor' => [], 'random_test' => ['led' => 'off'], 'water_pump' => ['pump' => 'off', 'threshold' => 30], // ← ADD THIS ];

Step 3: Add to the Registration Form

Open devices.html and add an <option> to the device type dropdown:

<select id="f-type" onchange="onTypeChange()"> <option value="random_test">Random Test (Demo)</option> <option value="chicken_coop">Chicken Coop</option> <option value="soil_sensor">Soil Sensor</option> <option value="ldr_sensor">LDR Sensor</option> <option value="water_pump">Water Pump</option> <!-- ADD THIS --> </select>

Step 4: Deploy

# Add the new file to deploy.sh FILES array, then: ./deploy.sh

That's it! The dashboard and device manager will automatically pick up the new module's sensors, controls, and code generation.

Troubleshooting

ESP32 Won't Connect to WiFi

  • Check SSID and password are correct: accounthack_fbnpa_2 / CLED02502F
  • Make sure the network is 2.4GHz (ESP32 does not support 5GHz)
  • Open Serial Monitor at 115200 baud to see connection logs
  • If stuck on "Connecting...", try pressing the BOOT button on the ESP32

API Returns 500 Error

  • Check the error message in the JSON response — it usually contains the exact SQL error
  • Most common: database credentials wrong in config.php
  • If "Table not found": the auto-creation in db.php should handle this, but you can manually import schema.sql via phpMyAdmin

Dashboard Shows "Offline"

  • The device must have sent data within the last 30 seconds to show as "Online"
  • Check that the ESP32 Serial Monitor shows successful POST responses
  • Verify the device_id in the Arduino code matches the registered device_id

Dashboard Shows No Data

  • Select a device from the dropdown first
  • The ESP32 must have sent at least one data point
  • Check browser console (F12) for JavaScript errors
  • Test the API directly: curl https://iot.aitalim.com/data/your_device_id

Code Generator Shows Wrong WiFi

  • WiFi credentials are defined in js/core.js
  • Update WIFI_SSID and WIFI_PASS constants, then redeploy

Upload Fails in Arduino IDE

  • Make sure the correct port is selected under Tools → Port
  • Try holding the BOOT button while uploading
  • If port doesn't appear, install the CP210x USB driver
  • On macOS, the port will be /dev/cu.usbserial-* or /dev/cu.SLAB_USBtoUART

Python Emulator Errors

  • Make sure you're running from the project root: python3 server_python_test/test_device.py
  • The emulator lists all devices first, then registers if needed, then starts the telemetry loop
  • Available functions: list_devices(), register_device(), send_telemetry(), fetch_config(), update_config(), read_history()
